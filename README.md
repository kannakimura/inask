# inask

社内ドキュメントから FAQ を自動生成し、自然言語で検索できる RAG（Retrieval-Augmented Generation）ツールのデモアプリケーションです。

> **このアプリはデモ用途で作成されています。**  
> 面接・ポートフォリオ・学習目的での利用を想定しており、本番運用には対応していません。

---

## 機能

- **ドキュメントアップロード**（PDF・テキスト・Markdown）
- **自動チャンキング＋ベクトル埋め込み**（Voyage AI）
- **FAQ 自動生成**（Claude API）
- **RAG 検索**：質問文を入力すると関連チャンクを検索し、Claude が回答を生成

## 技術選定理由

| 技術 | 選定理由 |
|---|---|
| PostgreSQL + pgvector | ベクトル検索専用DBを別途用意せず、既存のPostgreSQLに拡張機能を追加するだけでベクトル検索を実現。インフラをシンプルに保てる |
| Voyage AI (`voyage-3`) | OpenAI Embeddings と比較して日本語テキストの精度が高く、RAG用途での実績も多い。料金も安価でデモ用途に適している |
| Anthropic Claude | FAQ生成・回答生成の両方で使用。日本語の自然な文章生成が得意で、JSON形式の構造化出力も安定している |
| Redis + Laravel キュー | ベクトル化・FAQ生成は外部API通信を伴い時間がかかる。アップロードと非同期で処理することでUXを損なわない設計にした |

## 工夫した点

**処理の非同期化とポーリング**
アップロード直後にジョブをキューに積み、フロントは3秒ごとに自動リロードして「処理中 → 完了」をリアルタイムに伝える。JavaScript フレームワークなしでポーリングを実装した。

**チャンク分割**
ドキュメントをそのままベクトル化するのではなく、意味のまとまりで細かく分割してからベクトル化している。検索精度が上がり、回答の出典も段落単位で提示できる。

**冪等な Seeder**
`php artisan db:seed` を何度実行しても同じドキュメントが重複しない設計にした。デモのリセット・再構築がしやすい。

**config によるメッセージ管理**
UI の表示文言をすべて `config/messages.php` と `config/errors.php` に集約し、ハードコードを排除。将来の文言変更や多言語対応に強い構造にした。

**Policy による認可の集約**
管理者のみアップロード・削除できる権限チェックを `DocumentPolicy` に集約し、Controller・View に直接ロール判定を書かないようにした。

## 使い方

### ログイン

[http://localhost:8080/login](http://localhost:8080/login) を開いてください。

デモ用アカウントでログインします。

| 項目 | 値 |
|---|---|
| メールアドレス | `demo@innask.local` |
| パスワード | `password` |

---

### ドキュメントをアップロードする

1. ログイン後、ダッシュボードのアップロードフォームからファイルを選択
2. 「アップロード」ボタンをクリック
3. バックグラウンドで自動処理が始まります（ベクトル化 → FAQ 生成）
4. ステータスが「完了」になると FAQ が表示されます（ページは自動リロードされます）

対応ファイル形式：PDF・テキスト（.txt）・Markdown（.md）

---

### 社内ドキュメントを検索する

1. ナビゲーションの「検索」をクリック
2. 質問文を入力して「検索」ボタンをクリック
3. 関連ドキュメントをもとに AI が回答を生成します
4. 回答下部の「参照ドキュメント」から出典を確認できます

質問例：
- 「有給休暇の申請方法は？」
- 「経費精算の締め日はいつ？」
- 「入社初日の持ち物は？」
